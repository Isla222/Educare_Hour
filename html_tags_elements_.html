Hyperlinks

< a > < /a > < a href=”location of a document > Used to link one HTML page to another
< link > < /link > < link href=”location of a document” > Used to link an external resource in the HTML document

By default, links will appear as:

An unvisited link is underlined and blue
A visited link is underlined and purple
An active link is underlined and red

Input tag types

< input > element can be displayed in several ways, depending on its attribute

< input type=”button” > Displays a clickable button
< input type=”checkbox” > Displays a checkbox for selecting many choices
< input type=”date” > Displays an input field that should contain a date
< input type=”email” > Displays an input field for email addresses
< input type=”file” > Displays a button for choosing a file
< input type=”month” > Displays a month and year control which allows the user to choose whenever they want
< input type=”week” > Displays a week and a year control which allows the user to choose whenever they want
< input type=”password" > Displays a field for a password
< input type=”range” > Displays a control for entering a number range from 0 to 100 in default
< input type=”submit” > Displays a button for submitting the form
< input type=”tel” > Displays a field for telephone numbers
< input type=”text” > Displays a field for inputting a single-line text
< input type=”time” > Displays a control for seting the time
< input type=”radio" > Displays a radio button for selecting one of many choices
< input type=”reset” > Displays a reset button for resetting everything

Semantic tags

Describes its meaning to both the browser and the developer
Used to define different parts of a web page

< header > < /header > Defines a header for a document or a section, and represents a container for introductory content or a set of navigational links
< nav > < nav > Defines a set of navigation links
< section > < /section > Defines a section in a document, and is a thematic grouping of content, typically with a heading
< article > < /article > Defines an independent , self-contained content, and should be possible to distribute it independently from the rest of the website
< aside > < /aside > Defines content aside from the content like a sidebar and is considered separated from the content
< footer > < /footer > Defines a footer for a document or a section, and can have several tags of these in one document
< details > < /details > Defines additonal details that the user can open and close on demand
< summary > < /summary > Defines a heading for the < details > element
< figure > < /figure > Defines self-contained contents
< figcaption > < /figcaption > Defines a caption for < figure > element and can be placed as the first or as the last child of a < figure > element
